Projet – Traducteur

Introduction

Votre mission consiste à développer l'interface Web d'un traducteur en ligne entre plusieurs langues. L'opération de traduction proprement dite sera effectuée par des requêtes au service de traduction de Google.

Pour obtenir la note maximale, votre interface devra se comporter comme la démonstration ci-dessous :

Vous pouvez voir la démonstration dans une page indépendante.

Départ

Vous partirez d'une interface HTML statique qu'il faudra intégrer à votre application. Un canevas de cette application vous est aussi fournie.

L'application sait déjà s'initialiser avec un tableau de traductions et afficher une partie de l'entête de la page.

La fonction permettant d'interroger le service de traduction de Google est aussi présente.

Dans un dossier DAI/Projet/, enregistrez les fichiers traducteur.xhtml , dataTranslations1.js , googleTranslation.js et translator.js .

Pour info : L'interface a été construite avec Bootstrap 4 qui met à disposition un ensemble de classes CSS pour accélérer le développement et obtenir une apparence cohérente.

Évaluation

Seule la présence des fonctionnalités entrera dans l'évaluation de votre projet. La qualité du code ne sera pas prise en compte, sachant qu'un code de piètre qualité vous empêchera très rapidement d'ajouter de nouvelles fonctionnalités...

Voici les fonctionnalités qui seront évaluées :

  • Entête de la page
  • Onglets de navigation
  • Formulaire de requête
  • Tableau des traductions
  • Gestion de la pagination
  • Tri des colonnes
  • Affichage des onglets par langue
  • Suppression d'éléments

Distinction Affichage/Interaction

Pour chacune des fonctionnalités la distinction sera faite entre la capacité de l'application à afficher une partie de son interface graphique en fonction des données du modèle et sa capacité à réagir aux intéractions de l'utilisateur.

Par exemple, des points seront accordés si l'application sait afficher dans le formulaire de requête toutes les informations stockées dans le modèle (langues source et destination disponibles et sélectionnées) même si elle ne peut déclencher la demande d'une traduction.

Processus de développement

Il est impératif de comprendre la responsabilité de chaque composant de l'architecture SAM (actions, model, state, view) telle qu'elle a été détaillée en cours et explicitée dans les travaux pratiques.

Selon les principes de cette architecture, rappelons que :

  • L'interface Web déclenche des actions en leur fournissant des informations utiles ;
  • Les actions récoltent et préparent les données à présenter au modèle (model) pour qu'il se modifie en conséquence ;
  • Selon les données présentées, le modèle modifie ses propriétés concernées. Il est responsable de la conservation de la cohérence de ses propriétés. Seul lui y a le droit de les modifier ! (les autres composants peuvent toutefois y accéder en lecture) ;
  • Le composant state (état courant de l'application) dérive du modèle autant de propriétés qui seront nécessaires pour fabriquer l'interface graphique. Ces propriétés sont mises à jour à chaque passage à mesure des modifications du modèle ;
  • L'interface graphique est enfin construite par le composant view pour refléter les informations rassemblées dans model et state.

Le suivi scrupuleux de cette méthode vous garantira une structuration maintenable de votre application.

Conseil : Pour chacune des fonctionnalités de l'application, il faudra d'abord se concentrer sur la création ou la dérivation des données nécessaires à son affichage, avant de s'intéresser aux actions déclenchées par l'interface graphique.

Détail des fonctionnalités

Entête de la page

Pour l'instant, seul le bouton "À propos" est actif. Il permet d'afficher le nom du (des) auteur(s) du projet.

Onglets de navigation

Les onglets (tabs en anglais) doivent respecter les règles suivantes :

  • Le premier est toujours l'onglet "Traductions", le seul qui possède le formulaire de requête. Il affiche toutes les traductions, toutes langues confondues.
  • Quand l'onglet "Traductions" est sélectionné, les 3 onglets suivants affichent les langues possédant le plus de mots (colonnes "Depuis" et "Vers" cumulées).
  • Quand une langue est sélectionnée l'onglet correspondant s'affiche en deuxième position. Les 2 onglets suivants montrent les langues par ordre de mots disponibles décroissants.
  • Le nombre associé à chaque onglet renseigne du nombre de lignes disponibles pour chacun.
  • Le dernier onglet rassemble les langues non représentées dans les onglets précédents.

Formulaire de requête

Contraintes à respecter :

  • Les listes déroulantes "Depuis" et "Vers" ont une liste de langues distincte.
  • Le bouton d'échange (double flèches) permet d'inverser les langues. Il est toutefois désactivé dans les situations suivantes :
    • les langues "Depuis" et "Vers" sont les mêmes ;
    • quand une langue sélectionnée n'est pas présente dans l'autre liste.
  • Le bouton "Traduire" est désactivé quand les langues "Depuis" et "Vers" sont les mêmes.
  • L'appui sur le bouton traduire lance la traduction de l'expression qui sera ajoutée au tableau de traductions.

Attention : Il est possible que l'interrogation du service de traduction soit défaillante sur certains réseaux.
Dans ce cas, si on souhaite traduire le mot "pomme" de français en anglais et que le service de traduction est indisponible, on insérera la ligne ['fr', 'pomme', 'en', 'pomme'] dans le tableau de traduction.

Tableau de traductions

Le tableau affiche les traductions disponibles avec les caractéristiques suivantes :

  • Colonne 1 : indice de la traduction dans le tableau du modèle ;
  • Colonne 2 et 4 : le code des langues source et destination ;
  • Colonne 3 et 5 : les expressions originale et traduite ;
  • Colonne 6 : une case à cocher pour sélectionner la ligne ;

Notez qu'il faut pouvoir déterminer si une expression doit s'écrire de droite à gauche...

Gestion de la pagination

Pour un onglet, le nombre de pages dépend du nombre de lignes par page et du nombre de traductions disponibles.

À cela s'ajoute les considérations suivantes :

  • Les boutons "Précédent" et "Suivant" sont désactivés si on se trouve sur la première ou la dernière page respectivement.
  • Le choix d'une page n'affiche que les lignes de traduction correspondantes.

Tri des colonnes

Le contenu du tableau peut être trié en cliquant sur les entêtes du tableau.

Pour chaque colonne, le tri peut être croissant ou décroissant. Deux clics consécutifs sur la même entête inverse l'ordre de tri.

Un clic sur une autre entête rétabli le tri précédent pour cette colonne.

Affichage des onglets par langue

Un clic sur un onglet de langue affiche toutes les traductions concernant cette langue.

Pour ces onglets, il n'y a pas de formulaire de requête.

Les mots dans la langue sélectionnée apparaissent en début de ligne suivis de leur expression traduite correponsante.

Suppression d'éléments

Le bouton "Suppr. tous" demande une confirmation avant de supprimer toutes les traductions de l'onglet actif.

Le bouton "Suppr." devient actif dès qu'une case est cochée. Un clic dessus ne supprime que les lignes sélectionnées.
